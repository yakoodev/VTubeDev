# Подсказка для агента: Unity MCP

Это краткая подсказка, чтобы агент точно знал, как использовать единственный уже подключённый Unity MCP, какие команды выполнять и где брать результаты.

## 1. Какие MCP есть
- Всего один MCP: Unity. Он запущен через `docker-compose.yml` (`service: unity-mcp`). Сервер слушает 8080, LLM подключается к нему как к standard MCP.
- Дополнительно внутри Unity плагин `com.IvanMurzak.Unity.MCP.Installer` уже установлен, поэтому MCP-команды доступны сразу после запуска Unity и старта MCP сервера.

## 2. Команды для запуска и контроля
- `docker compose up --build --detach unity-mcp` — стартует MCP-сервер (ставим его в `tmp/` или `docker compose` из корня). Используй, если сервер не поднят.
- `docker compose logs --follow unity-mcp` — выводит логи сервера.
- `docker compose down unity-mcp` — останавливает сервер, когда тесты завершены.
- Если много задач, можно запустить `docker compose logs unity-mcp` и фильтровать по `ERROR`/`WARN`.

## 3. Использование внутри задач
- Перед стартом проверки поднимай MCP (`docker compose up`). Убедись, что Unity подключился и описал инструменты (информацию выводит лог `unity-mcp`). Если не подключился, попробуй перезапустить Unity или MCP-сервер.
- Когда Unity готов, запускай нужную сцену/прибор и делай необходимые скриншоты/видео (см. `docs/agent-hints.md`). Придерживайся согласованных имён файлов.
- Логи Unity можно взять из `%LOCALAPPDATA%\Unity\Editor\Editor.log` (через `powershell`). Отправляй выдержки в `tasks/...`.

## 4. Видео и аватар
- Видео ты добавишь вручную, поэтому агент должен только упомянуть путь `tasks/avatar/avatar-sim.mp4` (или альтернативный файл, когда появится) и описать, на какой задаче оно используется.
- Если всё же нужно новое видео, агент задаёт вопрос, но пока достаточно ссылки на существующий файл.

## 5. Что предложить дополнительно
Если не хватает информации, можешь попросить пользователя о:
- Больше MCP-клиентов (если надо переключиться с Docker на native `unity-mcp-server`).
- Примере команды, которая снимает скриншот/видео через MCP (если стандартный способ неудобен).
- Дополнительных тестовых сцен/данных для камеры (если текущего видео не хватает).

Эта подсказка помогает агенту не ошибиться с одноимённой MCP и быстро подготовить скриншоты/логи без вашего вмешательства.
