# D2_5 — /stream: Cache-Control + slow client policy

**Status:** todo

## Цель

Зафиксировать поведение для медленных клиентов и корректные заголовки кеша для MJPEG.

## Контекст / ссылки

- `docs/vtube-mvp.md` (Slow-clients, Cache-Control)

## Объём

**Включено:**

- `Cache-Control: no-store` для /stream
- Политика slow clients: дропаем старые кадры, отдаём свежий

**Не включено:**

- Адаптивное качество

## Шаги реализации

1. Добавить заголовок `Cache-Control: no-store` в /stream.
2. Реализовать буферизацию по принципу “последний кадр”.
3. Документировать поведение в коде.

## Критерии приёмки

- [ ] В ответе /stream есть `Cache-Control: no-store`.
- [ ] Медленный клиент не блокирует рендер, получает свежие кадры.

## Как проверить

- Открыть stream и проверить заголовок.
- Имитировать медленного клиента и убедиться, что кадры дропаются.
