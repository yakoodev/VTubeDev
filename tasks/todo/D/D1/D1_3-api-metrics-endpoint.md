# D1_3 — /api/metrics: базовые метрики

**Status:** todo

## Цель

Добавить endpoint `/api/metrics` для базовых метрик и диагностики (как в архитектурном холсте).

## Контекст / ссылки

- `docs/vtube-mvp.md`
- `docs/app-api.md`

## Объём

**Включено:**

- `/api/metrics` JSON
- Минимальный набор метрик (uptime, fps или frameCount, activeFormats, viewerCount если есть)
- Без тяжёлой телеметрии

**Не включено:**

- Prometheus формат
- История/агрегация

## Шаги реализации

1. Определить DTO ответа для метрик.
2. Подключить источники данных (where available).
3. Вернуть JSON и корректные коды.

## Критерии приёмки

- [ ] `/api/metrics` отвечает 200 и валидным JSON.
- [ ] Метрики обновляются во время работы.

## Как проверить

- Открыть `/api/metrics` и убедиться, что поля есть и меняются (uptime растёт).

## Артефакты

- Endpoint `/api/metrics`
- DTO/контракты
