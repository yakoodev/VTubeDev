# A4_1 — Миграции конфигов и versioning

**Status:** todo

## Цель

Реализовать правила миграции конфигов при изменении *Version.

## Контекст / ссылки

- `docs/vtube-mvp.md` (Versioning/Migrations)

## Объём

**Включено:**

- Проверка версии при загрузке
- Миграции между версиями
- Логирование факта миграции

**Не включено:**

- UI мастера миграций

## Шаги реализации

1. Ввести версию для ключевых конфигов.
2. Добавить миграции v1 → v2 (каркас).
3. Логировать обновления.

## Критерии приёмки

- [ ] Старый формат мигрируется автоматически.
- [ ] В логах видно, что миграция произошла.

## Как проверить

- Подсунуть старый JSON и убедиться, что он обновился.

## Артефакты

- Migration framework
