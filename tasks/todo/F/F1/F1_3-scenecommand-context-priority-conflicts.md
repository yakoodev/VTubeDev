**Status:** todo

## Контекст / ссылки

- `docs/vtube-mvp.md` (SceneCommand queue)

## Объём

**Включено:**

- Поля priority/actorId/timestamp
- Логирование конфликтов/отбрасывания

**Не включено:**

- Сложные политики разрешения конфликтов

## Шаги реализации

1. Расширить DTO SceneCommand.
2. В очереди учитывать priority.
3. Логировать конфликты и решения.

## Критерии приёмки

- [ ] Команды с большим priority применяются первыми.
- [ ] Конфликты видны в логах.

## Как проверить

- Отправить две конфликтные команды с разным priority и увидеть порядок.
