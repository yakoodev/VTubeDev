# F1_1 — SceneCommand: базовые команды + очередь применения на Update()

**Status:** todo


## Цель

Ввести единый язык SceneCommand и очередь применения, чтобы UI/Web/хоткеи не лезли напрямую в сцены.


## Объём

**Включено:**

- Команды SetData/SetOverlay/SetOutputProfile/SetInputSource
- Очередь
- Валидация


**Не включено:**

- Undo/Redo

## Контекст / зависимости

- Проверка через POST /cmd зависит от F1_2.




## Шаги реализации

1. Определить команды и их DTO.
2. Сделать очередь и применение в одном месте (Update).
3. Валидация команд и ошибки.


## Критерии приёмки

- [ ] Команды из web и UI работают одинаково.
- [ ] Невалидные команды не валят приложение.


## Как проверить

- Локально enqueue SceneCommand и увидеть применение в Update.
- Проверка через POST /cmd — после реализации F1_2.

- SceneCommand
