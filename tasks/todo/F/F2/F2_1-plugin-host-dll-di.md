# F2_1 — Plugin Host (DLL) + DI базовый контракт

**Status:** todo

## Цель

Реализовать загрузку DLL плагинов и базовый DI-контракт, как в архитектуре MVP.

## Контекст / ссылки

- `docs/vtube-mvp.md` (Plugins)

## Объём

**Включено:**

- Поиск DLL в глобальной и проектной папке
- Reflection load + инициализация
- Интерфейсы IPlugin / IPluginContext
- DI container регистрация сервисов

**Не включено:**

- UI управления плагинами
- Горячая перезагрузка

## Шаги реализации

1. Определить интерфейсы и контракт.
2. Загрузить DLL по списку путей.
3. Инициализировать плагины и зарегистрировать сервисы в DI.
4. Обернуть вызовы плагинов в try/catch.

## Критерии приёмки

- [ ] Плагин из DLL успешно загружается и инициализируется.
- [ ] Ошибка плагина не валит приложение.

## Как проверить

- Создать тестовый плагин, логировать init.
